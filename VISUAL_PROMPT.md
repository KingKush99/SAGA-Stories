# Visual Prompt

This app should look like a polished dark creator studio with bold, readable, image-first surfaces. Use the attached colorful tile reference as the quality bar: clear shapes, high contrast, strong icon silhouettes, consistent spacing, and confident blocks of color.

## Global Quality

- Prefer large, simple visual blocks over thin outlines and cluttered text.
- Keep buttons legible, tactile, and obviously clickable.
- Use strong contrast on dark backgrounds.
- Avoid tiny icons, low-opacity controls, hidden controls, and cramped spacing.
- Use real visual hierarchy: primary action at the bottom of the flow, navigation at the corners, details grouped into clear sections.

## Profile

- The avatar is a large circular image/mascot.
- The pencil edit icon sits visibly on the bottom-right edge of the avatar circle.
- Mailbox and settings are visible circular buttons in the top-right of the profile card, with mailbox immediately left of settings.
- The mailbox opens a thin dropdown directly under the mailbox button.
- Settings opens a square modal with clear options.
- Settings submenus open directly under the clicked option and stay within that one option column.
- Back buttons are red rounded rectangles with a clean white left arrow.
- X buttons are red circles with a white X and sit in the top-right.

## Mascots

- Provide 35 different CSS mascot choices plus one circular upload option.
- Mascots should be different animals with different color identities, not variants of one animal.
- Mascots use a black circular background and should feel like team-logo/avatar quality.

## Create

- Creation inputs belong on the left.
- Created audiobook/player output belongs on the right.
- Import, dictate, and AI-fill controls live inside relevant sections, not in the page header.
- Users can write manually, dictate into an idea/theme/outline field, import a manuscript, or ask AI to fill a section.
- Generate Book stays at the bottom of the creation flow.
- Generated books should match the selected page/word target as closely as possible.

## Live

- Live home shows image-first room cards, not open chat.
- Arrange categories as columns: Oldest / Longest on the left, Popular in the middle, New on the right.
- Cards should be colorful, image-heavy, and easy to click anywhere to enter.
- Inside a room, chat becomes Twitch-style with colorful usernames and dense live messages.
- Leaving with either Back or X stops audio immediately.

## Player

- Back button is top-left, X is top-right.
- 30-second controls show circular-arrow direction.
- Playback speed cycles from 0.5x to 2x and actually changes the timer.
- Chapter list opens and lets the user jump to a chapter.
- Remove sleep timer.
- Add sharing for current section and whole audiobook.

## Publish

- Publish is a vertical flow: Release container, Distribution container, then one large One-click Publish button at the bottom.
- There is only one One-click Publish button.
- Distribution cards are square, colorful, and navigated with middle-position arrows plus dot/page controls.
